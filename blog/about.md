# 关于本站

你好，我是梦泽。据某些技术论坛的标准，我是一名 PHP 全栈开发者；据我自己的标准，我不过是一个学历不高、半路出家、靠搜索框续命的人。这个博客是我用 VitePress 搭建的第四站——前三站，一个死鱼脏乱差我自己炸了，一站死于服务器到期，一站死于运营商跑路。我时常想，博客的寿命大约确实比我的耐心要短，但人总是健忘的，所以我又搭了这第三站，希望它能活得久一些，虽然"希望"这个词，我向来是不大信的。

## 个人简介

我是湖南人，09 年 2 月生，现居长沙。2023 年春天开始接触 Web 开发，从 HTML/CSS 手写静态页面起步，后来掉进 PHP 的坑，再往后顺藤摸瓜学会了 Vue、Node.js 和一些运维皮毛。如今主要在 NingZeStudio 维护开源项目，偶尔接点外包，所得仅够服务器开支和几杯咖啡。

说到技术栈，我"精通" PHP 现代化开发——"精通"二字是要加引号的，因为在任何技术论坛我都敢这么写，但在生产环境里遇到内存泄漏时，我通常先重启，再 Google，最后才承认可能是自己的问题。我也"熟练" Vue 生态，所谓熟练，意思是 npm install 很少报错，但 node_modules 体积超过 500MB 时，我会假装没看见。至于 K8s、Docker、微信小程序、Chrome 扩展……这些我都"浅尝"过。"浅尝"是一种委婉的说法，实际意思是文档看一半、Demo 跑通就觉得自己会了，真出问题先 Google 再重启，解决不了就假装没学过。

这种状态，我称之为"全栈"。听起来比"杂工"体面，也比"什么都不会"好听。世人大约都需要一些体面的称呼，我也不例外。

我还单独学过 UI/UX 设计，能在 Figma 里拖出精美且交互流畅的网页 Demo——配色和谐、间距讲究、动效丝滑，连 hover 状态都做了三种备选。然而一旦进入实际开发，这些设计就像超市购物车里的计划：进去时雄心勃勃，结账时只剩刚需。响应式布局开始崩坏，自定义字体加载失败，"稍微调一下"的按钮最终变成了内联样式堆叠的灾难。我至今不知道问题出在哪一步，可能是代码，可能是执行力，也可能是我对"实现"这个词有误解。Figma 里的完美和浏览器里的残缺，中间隔着的，大约是整个前端工程化的深渊。

## 写作

写博客的习惯始于 2023 年夏天。起初是为了记笔记防止遗忘，后来逐渐变成了一种整理思绪的方式。巅峰时期在 Typecho 上攒了三百多篇存稿，涵盖 PHP 踩坑、Linux 折腾、前端踩雷等各类主题，后因某小运营商突然倒闭，仅抢救回二十来篇，约 14 万字。这段经历教会我两件事：第一，数据备份很重要；第二，免费的东西最贵。然而我至今仍未学会备份，大约人的本性是难移的。

现在改用 VitePress 配合 Vercel，至少不用担心服务器跑路。写作风格比较随意，通常没有完整大纲，想到哪写到哪，偶尔技术，偶尔生活，偶尔纯粹的无病呻吟。如果你发现某篇文章前后矛盾，那很正常——人的想法本来就是流动的，正如我此刻写下的这段话，落笔前并无完整构思，写到哪算哪。

## 维护中的项目

### McLogs-Next
Minecraft 服务器日志分析工具，基于原生 PHP + FPM 开发，依赖 Aternos 服务链。代码写得颇为狼狈，但至今仍在勉强运行，支撑每日数千条日志解析请求。说它"勉强"，是因为现阶段的性能瓶颈主要在于 Aternos 的响应速度，而非我的代码——虽然我的代码，也确实没有什么优化余地。

- **McLogs-Next-API** — Headless API 核心服务。[https://github.com/NingZeStudio/McLogs-Next-API](https://github.com/NingZeStudio/McLogs-Next-API)

- **McLogs-Next-UI** — 官方 Web 界面，Vue 3 + Vite 构建，设计上追求简洁，实际使用中发现简洁的代价是功能藏得太深，用户时常找不到上传按钮在哪里。这大约是"极简"的另一种诠释。[https://github.com/NingZeStudio/McLogs-Next-UI](https://github.com/NingZeStudio/McLogs-Next-UI)

---

### LogShare2（预告）
McLogs-Next 的重构版本，基于 Hyperf + Swoole 重写，目标彻底摆脱 Aternos 依赖。目前处于开发阶段，API 骨架已搭好，核心解析逻辑还在和正则表达式搏斗。预计发布日期：待定。所谓"待定"，意思是取决于我什么时候能不拖延，而拖延这件事，我向来是擅长的。

### 其他项目

我开发了两个小项目，一个特点是优雅，另一个也是优雅。

- **[https://github.com/MornZe/StartPage](https://github.com/MornZe/StartPage)** — 基于 Vite + Vue 3 + TypeScript 的浏览器起始页。功能极简：一个搜索框，一组可自定义的快捷链接，以及根据时间自动切换的深浅色主题。没有天气组件，没有待办清单，没有一言 API。我觉得那些东西会分散你打开浏览器时的注意力——虽然你打开浏览器，本来就是为了分散注意力。这大约是自欺欺人的一种。

- **[https://github.com/MornZe/HomePage](https://github.com/MornZe/HomePage)** — 基于同样的技术栈，一个单屏滚动的个人主页。包含头像、简介、技能标签、项目列表和联系方式，所有信息都在首屏可见，无需滚动。我固执地认为，如果一个访客需要滚动才能了解你，那你可能写得太多，或者他没那么想了解你。这种固执，大约也是自我保护的一种。

- **LogShare** — https://logshare.cn/  
  初代 Minecraft 日志分享平台，目前仍在线上运行。LogShare2 开发中，虽然"开发中"这三个字，我已经说了很久。

  自用的浏览器起始页，极简到只有搜索框和时间显示。有时我会盯着它发呆，想不起自己要搜索什么，这大约是现代人的通病。
